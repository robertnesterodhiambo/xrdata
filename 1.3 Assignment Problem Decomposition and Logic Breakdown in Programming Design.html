Getting Started

A common approach in problem-solving is to analyze problems and divide a problem and its solution into several smaller modules at a high level. The approach is practical in programming design to break down complex programming problems into smaller, simpler, and manageable modules or classes. Modularization is one of the main principles of OOP software design for better quality, fast development, future reuse, and sharing.

Comprehending the requirements of any programming project or exercise is the first and essential step for any programming design and solution development.

You can figure out how to divide a problem and its solution into small modules, describe the entity (classes and attributes), and develop a method(s) and a control flow for the object-oriented programming solution. 

Graphs are primary and practical tools often used in programming design, solution development, and documentation. A UML diagram is a way to visualize systems and software. A graph like a flowchart or pseudocode can be an excellent choice to illustrate the control flow between program statements step by step. A flowchart is a diagram depicting a process, a system, or a computer algorithm. It is a diagrammatic representation of the solution to a given problem, but, more importantly, it provides a breakdown of the essential steps to solving the problem. A pseudocode is a way to express the logic of a program or algorithm without the constraints of a specific programming language's syntax.

A complete object-oriented graph representation and control flow represent the principles of OOP and involve structuring and conceptualizing the graph. A dependence graph shows the interrelationships among objects. The design documents can guide Java program source code development in the next step to its successful completion.

Upon successful completion of this assignment, you will be able to:

  • Analyze the problem and decompose the problem-solving process into procedures, functions, or modules.
  • Apply the principles of object-oriented programming (OOP) to design class in UML.
  • Illustrate and diagram problem-solving processes, classes, and methods in UML class diagram, UML activity diagram/flowchart and maybe sequence diagram and state diagram for all the class designs.

Java programming development in this course and any other programming development often requires you to choose unified modeling language (UML), flowchart, or pseudocode as tools to assist in developing Java classes and methods as preliminary and high-level designs for programming solutions first. Choosing graphical tools for high-level design can help with problem decomposition and Java programming design before Java programming and coding.
