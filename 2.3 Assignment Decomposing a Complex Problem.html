Getting Started

A common approach in problem-solving is to analyze problems and divide a problem solution into several smaller modules at a high level first and then focus on the detailed design for each sub-module. The approach is efficient in programming design to break down a complex programming problem into smaller, simpler, and manageable modules as a programming solution as a high-level programming design. Object Oriented Analysis and Design (OOAD) also focuses on the design of classes, attributes of the class or fields, and behaviors or methods. A class is like a blueprint. The object is the actual program component and has the class's behaviors. Each class defines a set of attributes (data members) and methods (functions or procedures).  

Comprehending the requirements of any programming project or exercise is the first and essential step before any program design – problem decomposition and modularization.  

Modularization is a process of decomposing a problem solution into sub-modules, and it is also a process of comprehending the problem for problem-solving. Modularization in Java programming design can help you understand the problem and its requirements and divide your programming solutions into sub-modules for better quality, fast development, future reuse, and sharing in a visualized tool like UML. 

Problem decomposition requires you to figure out how to divide a problem solution into smaller solution modules based on its functionality instead of one whole and lengthy solution and describe the entity (classes and attributes) and relationships to develop a control flow for the object-oriented programming solution of each module in UML.  

The unified modeling language (UML) defines modules or entities (classes) with attributes that support object-oriented programming (OOP) inheritance. Unique graphs can represent the attributes and relationships of entities in object-oriented programming (OOP). UML and Graphs used in programming design, solution development, and documentation are primary and practical tools. A graph like a flowchart or pseudocode can illustrate the control flow in a class method between program statements step by step.     

A complete object-oriented graph representation and control flow represents the principles of OOP and involves structuring and conceptualizing the graph. The graphical design documents will guide Java program source code development to its successful end.

Upon successful completion of this assignment, you will be able to:

  • Analyze a problem and decompose the problem into classes, objects, and methods.
  • Apply the principles of object-oriented programming (OOP) to design Java classes in UML.
  • Illustrate and diagram problem-solving processes, classes, and methods in UML class diagram, UML activity diagram/flowchart and maybe sequence diagram and state diagram for all the class designs.
